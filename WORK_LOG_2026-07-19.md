# Dashboard — work log, 19/07/2026

Admin panel (listing CRUD, uploads, auth). Everything below is **uncommitted**

## 1. App shell redesigned to the mockups

### Three-tone surface — `src/styles/global.css`

Added `--surface`. The shell sits on `--surface`, every panel on `--card`. In
light mode `--background` and `--card` are both white, so without a third tone
the sidebar and cards dissolved into the page. It inverts in dark: shell darker
than panels.

Verified: light shell L=96.5 / cards L=100 · dark shell L=2.0 / cards L=7.8.

### Sidebar — `src/components/nav_bar/index.tsx`

Brand block, `MENU` label, left-aligned items **top-aligned** (they used to be
vertically centred with the icons centred too), live count badge on Imóveis, dot
on Notificações, user footer with a working logout.

Two deliberate removals:

- **The collapse toggle is gone.** Neither mockup shows one and it didn't work —
  with the panel collapsed the class flipped to `w-[7.5rem]`, the CSS rule
  existed and read `width: 7.5rem`, but the computed width stayed 250px, and
  `min-w`/`max-w` didn't bind it either. I couldn't pin the cause down and
  stopped rather than ship a half-working control.
- **The whole sidebar no longer toggles on click** — that behaviour was why
  logout and every nav item carried `stopPropagation` hacks.

### Header — `src/components/page_header/index.tsx` (new)

Replaces the old global `Breadcrumb` (**deleted**). Route-aware: greeting + date
on the dashboard, breadcrumb + back arrow on add/edit, and it carries the single
"+ Adicionar". `NavBarItems` is now the right-hand cluster (search, bell, theme
toggle) and accepts children for the page's primary action.

Clock and theme both differ between server and client render — both are gated on
`useIsMounted()` (`src/hooks/useIsMounted.ts`, new), which uses
`useSyncExternalStore` rather than a mount effect so it costs one render, not
two, and doesn't trip the `set-state-in-effect` lint rule.

## 2. Dashboard page — `src/app/(content)/dashboard/page.tsx`

Mostly **real data**, not mock. Listing count, portfolio value, type donut,
district bars and "Adicionados recentemente" all come from the API on the *same
SWR key* the listing page and sidebar badge already use, so the page adds no
request of its own.

Only **Visualizações** and **Contatos WhatsApp** are invented, and both carry an
`exemplo` badge — as does Atividade, which uses real listing names with fake
timestamps.

**The growth chart has nothing to plot.** All 25 listings share a `createdAt` of
2025-10-25 (the bulk import), so over any six-month window the cumulative series
never moves and recharts draws a flat line pinned to the top of the panel, which
reads as broken. It shows "Sem movimentação no período" instead. The real chart
appears once listings land in different months.

## 3. Listing form — `src/app/(content)/real_estate/_components/real_estate_form/`

Four numbered step cards: Capa → Detalhes → Localização → Galeria.

New sub-components:

- `components/type_picker.tsx` — five icon buttons instead of a dropdown
- `components/segmented_field.tsx` — Venda / Aluguel / Venda e Aluguel

`components/preview_card.tsx` was **deleted** — the preview is now
`<RealEstateCard variant="preview">`, the same card the frontend ships, fed live
form values via a `thumbnail` override for the not-yet-uploaded blob.

**Salvar/Cancelar are pinned to the bottom**: the preview column is a fixed
shell, preview scrolls, actions don't.

Three deliberate divergences from the mockup:

1. **Kept the +/- steppers** — the mockup draws plain number inputs, but you
   explicitly asked to keep them.
2. **Kept Estado and Complemento** — the mockup drops them; the backend marks
   `state` required.
3. **Negociação has three options**, not two — the model enum includes `both`.

Prefix icons were dropped from the form fields to match the mockup; only `R$` on
Preço remains. Login still uses them.

## 4. Dropzone — `src/components/dropzone/index.tsx`

Rewritten. Takes an `existing?: string[]` prop, so **the edit page now shows the
saved cover** (badged "Capa atual", with a click-to-replace strip) **and the
saved gallery** (badged "Atual"). Verified against a real listing: 1 cover + 21
gallery images.

Preview after drop fills the box edge-to-edge instead of sitting inset in a grey
panel. Cover box 240px → 420px.

**Deliberate limitation:** existing images have no individual delete. The API
replaces the whole gallery on write, so a per-image X would be a promise the
backend can't keep. Needs a partial-gallery endpoint to do properly.

Also removed the two anime placeholder images from `public/test/` that were used
as drag feedback.

## 5. Bug fixed: `/real_estate` toolbar

`SelectPlain` passed its width to the inner Radix trigger, but `FieldWrapper`
wraps that in a `w-full` div — and a flex parent measures the **wrapper**. So the
width was ignored and both selects claimed an equal share of the row, squeezing
the search box to **111px**.

`FieldWrapper` now takes a `className` and sizing goes there. Search is 566px,
selects 220/200.

This was latent anywhere `SelectPlain` sits in a flex row, not just that page.

## 6. Shared card — `src/app/(content)/real_estate/_components/real_estate_card/`

same names. `preview` is used in the listing grid **and** as the form preview.

The grid's track was retuned `minmax(50rem)` → `minmax(32rem)`; the old width was
sized for the previous card and left a single column on a 1000px pane. Now three
columns of 320px, cards 340px tall — identical to the frontend.
